Step by step
How to Use the Editor

No installation, no account. Open the URL and you are already inside the editor. Here is everything you need to get started.

01

Choose a database from the sidebar

On the left side of the editor you will see two built-in databases. Click PixelMetrics to load a SaaS business analytics dataset — this is the same database used in Walter's LinkedIn Learning courses. Click Digital Evidence

You can also upload your own .sqlite or .db file by clicking the upload button in the sidebar. Your file stays local — it never leaves your browser.

💡 If you are following Walter's LinkedIn Learning course, start with PixelMetrics — it matches the course exactly.
02

Explore the schema before writing

Click any table name in the Schema Explorer to expand its columns and data types. This shows you what you are working with before writing a single line of SQL. Understanding the schema first is a habit senior analysts develop early — this makes it easy.

💡 Start with SELECT * FROM table_name LIMIT 10; to get a feel for the data before writing complex queries.
03

Write your query in the editor

Type your SQL in the main editor panel. The editor supports syntax highlighting, line numbers, and full SQL — SELECT, JOIN, GROUP BY, subqueries, window functions, everything SQLite supports.

Run your query by clicking the Run button in the toolbar or using the keyboard shortcut.

⌨️ Keyboard shortcut: Ctrl + Enter on Windows · Cmd + Enter on Mac
04

Review your results

Results appear in the panel below the editor. Drag the divider between the editor and results panel to resize each section — give yourself more room for either the query or the output depending on what you need.

The row count appears in the top right of the results panel. If your query returns an error, the error message displays there with enough detail to diagnose what went wrong.

05

Click Review to get AI feedback

After running your query, click the Review button. The AI reads your query and the full database schema, then returns three things: whether your query is correct, a note on style and readability, and one concrete improvement tip.

The feedback appears in the right panel. It is written in plain language — not jargon — and designed to teach, not just correct.

💡 Use Review even when your query works. Correct is the floor, not the ceiling — style and efficiency matter in real jobs.
06

Export your query or results

Click ↓ Query to download your SQL as a .sql file. Click ↓ Results to download the output as a .csv file — ready for Excel, Tableau, or any other tool you use for further analysis.

Interface overview
Know Your Workspace

Left Sidebar

Database selector, file upload, and Schema Explorer. Click any table to expand its columns and data types. Always visible so you can reference structure while writing.

Main SQL writing area with syntax highlighting and line numbers. The Run and Review buttons live in the toolbar above. Drag the horizontal divider to resize the editor and results split.

Results Panel

Query output with row count. Displays tables, error messages, or a confirmation if your query ran successfully but returned no rows. CSV export button available here.

Common questions
FAQ
Do I need to create an account to use the editor?
No. Open the URL and the editor loads immediately. You will be asked for your email to join the WSDA community, but you can click through directly to the tool without signing up.
What version of SQL does the editor support?
The editor runs SQLite — which covers the vast majority of SQL used in data analyst roles. SELECT, JOINs, GROUP BY, HAVING, subqueries, CTEs, and window functions all work. Dialect-specific syntax from PostgreSQL or SQL Server may differ slightly.
Can I upload my own database?
Yes. Click the upload button in the sidebar and select any .sqlite or .db file from your computer. The file loads locally and never leaves your browser.
Is this connected to Walter's LinkedIn Learning course?
Yes. The PixelMetrics database is the exact dataset used in Walter's LinkedIn Learning SQL courses. If you are following along with the course, use PixelMetrics to practice the same queries covered in each module.
How accurate is the AI feedback?
The AI reviews your query against the actual schema and evaluates correctness, style, and efficiency. It is accurate for standard SQL patterns. For highly complex queries, treat the feedback as one perspective — always test your query logic against expected results.
Is this tool free forever?
Yes. The AI SQL Editor is a free tool from Walter Shields Data Academy. There is no paid tier, no trial period, and no feature gating.
